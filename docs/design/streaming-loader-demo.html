Piano Helper - OMR streaming loader

The score renders system by system as the engine recognizes it, top to bottom. Finished systems show crisp real notes; the system being decoded right now gets an "active" treatment; systems still pending below show an animated loading state on empty staves (no fake notes). Toggle the concepts and watch a fake 6-system score go from all-pending to all-done on a ~30s loop.

Concept
System 0 / 6

done (real notes) decoding now (active) pending (loading, no fake notes)

Timing mirrors prod CPU: a ~28s blank lead-in, then a system finalizes about every ~9s. This demo compresses that to a ~30s loop (lead-in ~5s, ~4s per system) so the motion is reviewable; "Fast preview" compresses further. Attachment point for the real build: an overlay layer inside #sheet, one absolutely-positioned box per <g class="system"> bounding box, marked done / active / pending from the stream's k of total. CSS animation only (transform + opacity), no layout thrash.
